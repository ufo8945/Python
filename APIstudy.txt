API (Application Programming Interface)

 

ㅁ API란?

- 위키의 말을 빌리자면, 응용 프로그램 프로그래밍 인터페이스로 응용 프로그램에서 사용할 수 있도록, 운영 체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 만든 인터페이스이다.

- 위키의 말이 너무 어렵다.

- 스마트폰의 UI(User Interface) 중 홈ㅉ 버튼을 떠올려보자. 홈 버튼은 사용자들이 홈 화면으로 돌아갈 수 있게끔 도와주는 수단이다. 즉, 스마트폰(HW)과 사용자(USER)를 이어주는 매개체이다.

- API도 같은 맥락으로서 프로그램과 또 다른 프로그램을 연결해주는 일종의 다리라고 볼 수 있다.

ㅁ API를 활용해서 할 수 있는 것

- 일기예보 정보를 자신이 만든 웹 페이지에 띄우려면?

- 지도를 이용한 웹 서비스를 제작하고 싶다면? (길 찾기, 맛집 찾기)

- 자신의 웹 서비스가 사용자들로 하여금 결제가 가능하도록 만들려면?

 

ㅁ 위의 예시로 위키의 API 뜻 다시 이해 해보기

- 운영체제나 프로그래밍 언어가 제공하는 기능 : Naver의 날씨정보, Daum의 지도정보, Kakao의 카카오페이

- 응용 프로그램 : 우리가 만든 서비스(웹 페이지, 웹 서비스 등)

- 즉, API는 내가 만든 서비스가 개인 개발자, 기업, 기관이 제공하는 기능, 프로그램을 사용할 수 있게끔 도와주는 중간 매개체이다.

 

ㅁ 공공 API

- “공공데이터포털” : 국가에서도 공공 API를 제공하고 있다.

- “직방” 어플도 국가공공 API를 토대로 만들어진 서비스이다.

 

ㅁ API의 역할

- 서버와 데이터베이스에 대한 출입구 역할 : 접근이 허용된 사람들만 접근성 부여

- 모든 접속을 표준화 : 기계/OS 상관없이 누구나 동일한 액세스를 얻을 수 있다.

 

ㅁ API유형

- private API : 내부 API로, 제 3자에게 노출되지 않는다.

- public API : 개방형 API로, 누구나 제한 없이 API를 사용할 수 있다.

- partner API : 기업이 데이터 공유에 동의하는 특정인들만 사용할 수 있다.
